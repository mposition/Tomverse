# Tomverse AI 사용자 흐름 QA 설계

## 목적

공식 상업용 출시 전에 실제 사용자 관점으로 핵심 흐름을 검증하고, 기능 실패뿐 아니라 클릭 혼동, 상태 피드백 부족, 반응형 레이아웃 문제와 접근성 마찰을 발견해 작은 단위로 개선한다.

## 기본 원칙

- 로컬 환경에서 기능과 UI를 먼저 안정화한다.
- 운영 환경에서는 OAuth, R2, 실제 AI 공급자, 공개 공유 URL처럼 로컬에서 동일하게 재현하기 어려운 항목만 최종 검증한다.
- 공통 기능은 한 번 정의하고 PC와 모바일에서 각각 표현 및 상호작용을 검증한다.
- 발견 사항은 재현 조건, 기대 결과, 실제 결과, 심각도, 화면 환경을 함께 기록한다.
- 수정 후 관련 흐름과 기존 핵심 흐름을 모두 재검증한다.

## 대상 환경

### PC

- Chrome: 1920x1080, 1366x768
- Edge: 1920x1080, 1366x768
- 키보드 전용 탐색과 마우스 상호작용

### 모바일

- iPhone Safari 기준: 390x844
- Android Chrome 기준: 412x915
- 가상 키보드, safe area, 터치, 스와이프, 화면 회전

### 반응형 경계

- Tailwind `md` 브레이크포인트 바로 아래와 바로 위
- 모바일 및 데스크톱 shell 전환 시 기능과 상태 유지 확인

## QA 방식

혼합형 QA를 사용한다.

- 자동화: 반복 가능한 핵심 사용자 흐름과 회귀 검증
- 수동 검증: 사용감, 클릭 혼동, 레이아웃, 가상 키보드, 실제 모바일 Safari 동작
- 운영 검증: 외부 서비스 연동과 배포 환경 차이

## 사용자 흐름

### 공통 흐름

1. 게스트로 진입하고 언어를 변경한다.
2. 로그인하고 사용자 설정이 복원되는지 확인한다.
3. 새 대화를 시작하고 기본 모델을 확인한다.
4. 모델을 최대 3개까지 선택하고 제한 안내를 확인한다.
5. 메시지를 전송하고 모든 모델의 사용자 메시지와 응답 상태를 확인한다.
6. 특정 모델에만 추가 질문을 보낸다.
7. 이미지, PDF, Office 파일을 선택, 붙여넣기 또는 드롭 방식으로 첨부한다.
8. 대화를 잠그고 검증 전후의 접근 권한을 확인한다.
9. 공유 링크를 생성, 열람, 취소하고 읽기 전용 상태를 확인한다.
10. 개별 대화와 전체 대화를 다운로드한다.
11. Private Mode에서 서버 저장 여부와 안내 문구를 확인한다.
12. 로그아웃 후 게스트 상태와 제한이 올바르게 적용되는지 확인한다.

### PC 전용 흐름

- 다중 모델 패널의 크기, ON/OFF, 교체, 제거
- 입력창 및 팝업의 위치와 겹침
- 파일 드래그 앤 드롭 및 브라우저 기본 파일 열기 방지
- 팝업 focus trap, Escape 닫기, 방향키 및 Tab 탐색
- 1366px 화면에서 사이드바, 패널, 입력창의 최소 사용성

### 모바일 전용 흐름

- 대화 목록 drawer 열기, 닫기, focus trap
- 모델 탭과 좌우 스와이프로 활성 모델 전환
- 메시지 전송 직후 현재 화면에 즉시 렌더링
- 가상 키보드 표시 중 입력창과 전송 버튼 접근
- iOS Safari의 가로 넘침, 확대, safe area
- 터치 대상 크기, 스크롤 충돌, 화면 회전 후 상태 유지

### 교차 기기 흐름

- PC에서 생성한 대화를 모바일에서 열고 이어서 전송
- 모바일에서 생성한 대화를 PC에서 열고 다중 패널 상태 확인
- 잠금, 공유, 제목, 선택 모델 상태의 기기 간 일관성 확인

## 판정 기준

각 시나리오는 다음 기준으로 판정한다.

- 기능: 요청한 작업이 정확하게 완료된다.
- 피드백: 로딩, 성공, 실패, 제한 상태가 즉시 이해된다.
- 효율: 주요 작업이 불필요한 왕복 없이 완료된다.
- 안정성: 중복 클릭, 빠른 모델 전환, 느린 네트워크에서도 상태가 깨지지 않는다.
- 레이아웃: 겹침, 잘림, 예상치 못한 화면 이동이 없다.
- 접근성: 키보드, focus, ARIA 상태, 명암 대비와 터치 영역이 적절하다.
- 국제화: 한국어, 영어, 중국어에서 문자열 누락과 레이아웃 파손이 없다.

## 발견 사항 처리

- Blocker: 로그인, 메시지 전송, 데이터 접근 또는 결제가 불가능함
- High: 주요 기능 실패, 데이터 노출, 메시지 유실, 반복 충돌
- Medium: 우회 가능한 기능 오류, 뚜렷한 UI 혼동, 접근성 실패
- Low: 문구, 간격, 미세한 시각적 불일치

수정은 가능한 한 하나의 원인과 하나의 사용자 흐름 단위로 제한한다. 수정 후 lint, TypeScript 검사, production build와 해당 시나리오 재검증을 수행한다.

## 자동화 범위

Playwright 기반으로 다음 흐름부터 자동화한다.

- 게스트 첫 진입과 언어 변경
- 새 대화 및 모델 선택 제한
- 메시지 전송과 모델 탭 또는 패널 전환
- 대화 drawer 및 주요 팝업의 키보드 동작
- 반응형 주요 뷰포트의 가로 넘침 검사

OAuth 공급자 로그인, 실제 AI 답변 품질, R2 업로드와 모바일 Safari 실기기 동작은 운영 또는 수동 검증으로 유지한다. 자동화에서 실제 외부 AI 호출은 비용과 비결정성을 피하기 위해 응답을 제어한다.

## 운영 환경 최종 확인

로컬 QA가 통과한 뒤 `https://tomverse.app`에서 테스트 계정으로 다음만 확인한다.

- OAuth 로그인과 계정 연결
- R2 업로드, finalize, 첨부 메시지 처리
- 활성화된 모델별 실제 API 응답
- 공유 URL의 public origin, noindex, 취소 및 만료
- 보안 헤더, CSP 보고 URL, rate limit 사용자 안내

운영 데이터와 실제 사용자 대화는 변경하지 않으며, QA용 대화와 첨부파일은 테스트 후 정리한다.

## 완료 조건

- Blocker와 High 발견 사항이 0개다.
- Medium 발견 사항은 수정되거나 출시 판단과 함께 명시적으로 수용된다.
- PC와 모바일 핵심 흐름이 모두 통과한다.
- 운영 환경 최종 확인이 통과한다.
- 자동화된 핵심 회귀 테스트와 기존 `npm run check`가 통과한다.
