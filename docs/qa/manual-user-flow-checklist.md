# Manual User Flow Checklist

Use this checklist for real-browser and real-device launch sign-off after the automated Playwright suite passes.

Completion criteria:

- Blocker and High findings must be zero before launch.
- Medium findings must be fixed or explicitly accepted before launch.
- Every environment section must have an owner, result, and date.
- Evidence should link to screenshots, recordings, logs, or issue IDs when a flow fails or needs visual sign-off.

Result values: `Not run`, `Pass`, `Fail`, `Accepted`.

Severity values: `-`, `Low`, `Medium`, `High`, `Blocker`.
